# SituationMonitor

Go service that aggregates news (RSS), X/social data (Grok sweeps via OpenRouter), and market quotes into a local dashboard served over Tailscale.

## Architecture

- Single-process Go binary with independent worker goroutines (RSS, sweep, translate, market)
- SQLite database at `./data/situation.db`
- HTTP server at `127.0.0.1:8080` (configurable via `LISTEN_ADDR`)
- All config via environment variables (see `internal/config/config.go`)

## Daily Briefings

Daily briefing pages are generated by Claude via the `/daily-briefing` skill and written as standalone HTML files to `data/pages/YYYY-MM-DD.html`. The Go server serves these at `/daily/{date}` with an index at `/daily/`.

The skill fetches data from the local APIs, does web searches for additional context, and produces a polished intelligence-style briefing.

## Key paths

- `cmd/situation-monitor/main.go` — entry point
- `internal/httpserver/` — HTTP routes and templates
- `internal/store/` — database access layer
- `internal/openrouter/` — Grok sweep client (X/social data collection)
- `config/brief.txt` — sweep focus areas
- `config/feeds.txt` — RSS feed URLs
- `data/pages/` — generated daily briefing HTML files
- `.claude/skills/daily-briefing.md` — Claude skill for generating briefings

## Build & run

```bash
go build -o situation-monitor ./cmd/situation-monitor
./situation-monitor  # reads .env
go test ./...
```
